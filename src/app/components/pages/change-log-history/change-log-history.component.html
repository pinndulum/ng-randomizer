Synthetic Dataset Replay

This page models non-production database population of synthetic change-history records. A fresh refresh creates new mock data for demos, QA, or local development without relying on production data. Turn on replay when a bug report, training walkthrough, or repeatable seed-data script needs the same generated rows again. With the same template, options, and seed, mock-randomizer regenerates the same dataset instead of drifting between runs. When date fields use relative values such as now or today-7D, pair the seed with withClock so time-sensitive values replay too.
